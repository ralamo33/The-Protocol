Core Architecture

Anomaly and Principal are the two load-bearing data entities. All other data attaches to one or both. Both can own unique Experiments and a Management protocol. Anomalies persist in the facility across cycles. Profile is tracked per-anomaly and also persists.

Resources — GameState values
COHESION hidden
Combined moral erosion + supernatural decay. At 0 → INHUMAN ENDING. Drives narrative tone and hidden roll penalty.
EFFECTIVENESS hidden
How well containment is working. Starts high, decays to 0 → CONTAINMENT FAILURE. Merges pressure + external fear into one clinical measure.
BUDGET shown
Only visible resource. Spent on management protocols. Creates hard choices.
CAPABILITY hidden
Accumulated research. Sliding number with distinct tiers. Unlocks options silently — player discovers by seeing new choices appear.
PROFILE hidden per-anomaly
Entity knowledge as a percentage. 100% = file complete. Persists alongside the anomaly across cycles. Earned by running study experiments.
CYCLE hidden
Internal counter. Gates certain anomaly appearances (KETER in cycles 5–6). Player infers progression from context.
DIRECTOR INTEGRITY hidden
Director's own degradation. At low levels, log language becomes cold: "3 personnel resources lost" instead of "3 people died."
PERSONNEL derived
Three separate pools: Researcher, Security, Engineer. See below.
Data — TOML files, loaded at startup
ANOMALY
Title + brief only shown. Persists in facility across cycles. Owns 2 unique experiments + 1 management protocol. Profile tracked per-instance.
PRINCIPAL "Principal Scientist" at intro; "Principal" in-game & code
Named lead characters. Own 1 signature experiment + 1 management protocol. Can die, transform, or be incapacitated.
EXPERIMENT
Title + description shown (hints only — no mechanics visible). Logic in co-located .py. Data in .toml. Path: experiment/slug/name.{toml,py}
MANAGEMENT protocol chosen before experiment
3 per cycle: 1 standard, 1 principal's, 1 anomaly-specific. Shapes experiment conditions and outcomes.
EVENT was: Incident
A conditional game-loop interruption. Not inherently positive or negative. May branch into subtypes over time.
ENDING
3 data entries. Trigger conditions, final screen text, tone.
COMMUNICATIONS was: Narrative Bank
Template line pools per context. No-repeat queues. One file per pool.
ASSET provisional was: Equipment
Obtained obscurely through experiments. Description cryptic. Effects surface as new options — player discovers, doesn't receive.
Personnel Pools — 3 categories, assignment model

Start unassigned each cycle. Assigning to experiments or management protocols depletes the pool even when things go well. Incidents shrink pools permanently.

CategoryDepleted by
Researcher Assignment to study/analysis experiments.
Security Assignment to management protocols involving physical threat.
Engineer Assignment to advanced management protocols requiring infrastructure work.
